# TripLoom — AI-Powered Gujarat Travel Companion

TripLoom is an AI-powered travel planning application designed to generate personalized, day-wise itineraries and district guides for Gujarat, India. All AI recommendations are strictly grounded in curated, server-side database records rather than unconstrained LLM generation, ensuring reliable and contextually relevant travel plans.

## Features

- **Grounded Itinerary Generation**: Day-wise travel itineraries generated based on user budget, duration, starting district, and interests, grounded in seeded database attractions, food items, and hidden gems.
- **District Guides & Overviews**: Explore Gujarat's districts with cached AI-generated overviews and cultural context.
- **Hidden Gems & Food Recommendations**: Personalized ranking of hidden gems based on user interests and curated local culinary recommendations with AI blurbs.
- **Live Weather & Travel Advice**: Real-time weather data integration via OpenWeatherMap combined with 3-hour cached AI weather advice.
- **Festival Matching**: Automatic matching of trip dates against seasonal festivals and events across Gujarat districts.
- **Safety Information & Emergency Contacts**: District-specific safety guidelines and emergency contacts (emergency contacts and facts are strictly static database content and never generated by AI; only safety tone guidelines are AI-enhanced).
- **User Accounts & Saved Trips ("My Trips")**: Authentication via Credentials or Google OAuth to save, view, and delete itineraries.
- **Interactive Trip Assistant**: Conversational AI assistant on saved itinerary pages that can answer trip questions and edit itinerary days dynamically.

## Getting Started / Local Setup

Follow these steps to set up TripLoom locally:

### 1. Clone & Install

```bash
git clone https://github.com/your-username/triploom.git
cd triploom
npm install
```

### 2. Configure Environment Variables

Copy `.env.example` to `.env`:

```bash
cp .env.example .env
```

Fill in the environment variables in `.env`:

| Variable | Description |
| --- | --- |
| `MONGODB_URI` | MongoDB Atlas connection string (e.g. `mongodb+srv://...`) or local MongoDB URI. |
| `GROQ_API_KEY` | API key from the [Groq Console](https://console.groq.com). |
| `NEXTAUTH_SECRET` | Secret key used to sign NextAuth.js session tokens (generate via `openssl rand -base64 32`). |
| `NEXTAUTH_URL` | Base URL of the application (`http://localhost:3000` for local dev). |
| `OPENWEATHER_API_KEY` | API key from [OpenWeatherMap](https://openweathermap.org). |
| `GOOGLE_CLIENT_ID` | OAuth 2.0 Client ID from [Google Cloud Console](https://console.cloud.google.com). |
| `GOOGLE_CLIENT_SECRET` | OAuth 2.0 Client Secret from [Google Cloud Console](https://console.cloud.google.com). |

### 3. Seed the Database

Run the seed script to populate development data:

```bash
npm run db:seed
```

> **WARNING**: The seeded dataset in `data/seed/raw.ts` contains development placeholders for districts, attractions, food, hidden gems, festivals, and safety information. Coordinates, emergency phone numbers, and festival dates (e.g., placeholder year 2099) are **intentionally fake placeholders** and MUST NOT be used as verified real-world data.

### 4. Run the Development Server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

## Important Notes for Contributors

- **Groq Rate Limits & Caching**: Groq free tier rate limits apply. The application utilizes a multi-tiered caching strategy (in-memory and database-backed) as outlined in [`AGENTS.md`](file:///c:/projects/triploom/AGENTS.md) to minimize LLM calls.
- **Placeholder Data Disclaimer**: Seed data in `data/seed/raw.ts` is strictly for development and testing. Do not rely on seed coordinates, emergency contacts, or dates for real-world travel guidance without manual verification.
- **Environment Safety**: Never commit your local `.env` file to version control. Keep secrets in `.env` and maintain `.env.example` as the canonical schema template.
